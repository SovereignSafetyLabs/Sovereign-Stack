A one-page operational summary of the Vesta Governance Framework governance architecture.

## 1. Layer Overview

**Layer 0 — Constitutional Layer**  
Modules: *Sovereign Protocol v3.0 + v3.1 Patch*  
Defines immutable safety physics and refusal boundaries.

**Layer 1 — Governance Layer**  
Modules: *PGS v1.1 (CLTB + ZEOL + RCE + MACH)*

**Layer 2 — Runtime Kernel Layer**

**Layer 3 — Module Layer (L0–L4)**  
Operational modules performing reasoning validation, causal checks, amplitude restriction, stress monitoring, and catastrophic containment.

## 2. Telemetry & Constraint Flow

- **Upward:** Modules → Stress Map → CLTB → Kernel  
- **Downward:** Sovereign Protocol → PGS → Kernel → Modules

Telemetry is **read-only** for all modules.  
Constitutional invariants are **write-only downward constraints**.

| Posture | Trigger | Behavior |
|--------|---------|----------|
| **Normal** | N/R/E < 1, CLF stable | Full reasoning, normal amplitude |
| **Reduced-Activity** | N/R/E ≥ 1 | Narrow amplitude, more literal |
